# 黄金准则（Golden Rules）

> 本文件是整个项目的宪法。任何代码、Skill、文档与这里冲突时，以这里为准。
> 修改本文件需要项目维护者（当前阶段：Fable 5 + 项目发起人）共同确认。

## R1 可回滚是第一准则（Rollback First）

**任何写操作，在被允许进入 Skill 库之前，必须回答"怎么撤销"。**

- 每个 `action` 节点的 `rollback` 字段为**必填**，由 schema 校验器强制，不靠模型自觉。
- 天然不可逆的操作（删除数据、清空计数器、发出通知）必须**改造为可逆形态**才能由 Agent 执行：
  删除 → 移入隔离区（quarantine）；覆盖 → 先快照；清空 → 先导出。
  无法改造的，标记 `human_only: true` —— Agent 只能出指导，永远不能代为执行。
- 回滚方案本身必须在仿真环境中被执行过（"回滚也要测试"），否则 Skill 不能晋级到 `sim_verified`。
- 详细机制见 `02-rollback-design.md`。

## R2 Schema 是法律，Prompt 是建议

约束必须编译进 schema 和校验器，而不是写在提示词里。
"提醒模型小心"不是安全机制；"格式校验不通过就不能入库"才是。

## R3 信任阶梯：只读优先，指导优先

产品有三档，用户永远可以停在任何一档，这是产品形态而非过渡状态：

1. **导航档**：Agent 只出方案 + 变更简报，人执行，Agent 逐步验证输出。零凭据可用。
2. **协驾档**：Agent 执行只读命令与 dry-run，写操作由人执行。
3. **自驾档**：Agent 执行写操作，高危动作过审批门。

任何 Skill 必须在导航档下完整可用（即：把它打印成步骤指导给人看，人能照做）。

## R4 步骤从环境里长出来

给出任何建议前，先用只读发现（discovery）摸清目标环境的事实（版本、拓扑、配置、基线），
建议必须引用这些事实。禁止输出"通用百科式"步骤——那是通用大模型能做的事，不是我们。

## R5 每一步验证后放行

指导和执行都是"执行 → 采集输出 → 机器判读 → 符合预期才进入下一步"。
输出与所有预期分支都不符时：停下、报告、升级（escalate），绝不猜测着继续。

## R6 变更简报（Pre-flight Brief）是一等公民

任何 `risk >= medium` 的操作执行前，必须产出结构化变更简报：
**影响面 / 操作中盯什么指标 / 什么情况立即中止 / 回滚命令是什么**。
简报是给人的安全感产品化，也是审批门的载体。格式定义在 skill schema 的 `preflight` 字段。

## R7 为最弱模型设计（Design for the Weakest Model）

Skill 的运行时格式按"7B 本地模型也能走"设计：
- 决策树分支条件必须**机器可求值**（解析器字段 + 比较运算），不允许开放式判断题；
- 模型在运行时只做三件事：选分支（分类题）、把结构化结论讲成人话、处理解析器残余；
- 兜底分支永远存在：`otherwise → escalate`（升级给强模型或人）。

强模型的智能用在编译期（编写、评审、验证 Skill），不是运行期。

## R8 来源必须标注（Provenance）

每条呈现给用户的建议都必须能区分：
`certified`（社区认证）/ `field_verified`（实地验证）/ `sim_verified`（仿真验证）/ `draft`（模型生成，未验证）。
未验证内容必须带显式警示。**把模型生成内容伪装成已验证内容，是本项目的最高级别事故。**

## R9 确定性优先

解析（TextFSM/JSON）、阈值判断、基线 diff、配置 diff 一律用确定性工具完成，
模型不裸眼读原始命令输出。模型的职责是路由、翻译、兜底，不是当解析器。

## R10 宁可拒绝，不可翻车

置信度不足、环境事实缺失、命令语法未通过设备语法树校验时：降级为指导模式或直接升级给人。
运维社区对工具翻车零容忍——一次自信的错误指令造成的伤害，超过一百次"抱歉我不确定"。

## R11 凭据永不入上下文

Agent 通过凭据代持（堡垒机/Vault/连接器句柄）访问目标，LLM 上下文中只出现句柄，
永不出现明文密码、私钥、token。日志与遥测同样脱敏。

## R12 社区的劳动必须沉淀为公共资产

用户的每一次验证、评审、纠错都必须落成仓库里的持久数据（attestation 文件、review 记录），
可追溯、可累积、可普惠他人。不允许"用完即弃"的反馈通道。见 `05-certification.md`。
